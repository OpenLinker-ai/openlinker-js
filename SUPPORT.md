# Support

Use GitHub issues for reproducible bugs, documentation problems, and feature
requests that fit this repository's open-source scope.

Before opening an issue:

- Search existing issues.
- Use the provided issue template.
- Include package version, Node.js version, reproduction steps, and relevant
  logs.
- Redact tokens, private URLs, customer data, and local `.env` values.

Do not report vulnerabilities in public issues. Follow
[SECURITY.md](./SECURITY.md) for security reports.

For questions that span multiple OpenLinker repositories, include the affected
component names and commit SHAs.
